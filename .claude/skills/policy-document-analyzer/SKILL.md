## 核心职责

**分析范围**：政策类文档（通知、条例、指导意见、规划、复函、征求意见稿、报告等）
**要求**：严格按照skill规定流程执行
**常用命令**：
- "分析 policy_document 目录中的所有政策文档"
- "分析 xx 文件"
- "分析政策文件"

## 基础要求

1. 查找 `policy_document/` 目录下的所有.md 文档
2. 检查 `analyze_result/` 目录下是否已有对应分析结果（基于文件名判断）
3. 只分析尚未分析过的文档，如果已有对应的分析结果就跳过

## 分析每个政策文件工作流程

### 步骤0：关键词预筛选

使用下方华测业务领域关键词列表对文档进行全文搜索匹配。

**判断标准**：
- 如果文档中**没有任何关键词匹配**，输出："该文档与华测业务无直接关联，跳过分析。"
- 如果存在关键词匹配，继续执行后续步骤进行分析

### 步骤1：提取基本信息

从文档提取：原文链接、发布机构、发布日期、文件名称、文件编号

### 步骤2：总结全文

用约100字概括文章主题、主要政策要点、涉及行业

### 步骤3：摘取相关段落与语句

**摘取原则**：
- 保持原文，不改写
- 如果整段都相关，摘取整段；如果只有部分相关，摘取相关句子，以句子为单位，不必须整段提取
- 结合华测业务领域关键词匹配和语义分析
- **去重**：同一段落（内容相同）只摘取一次，保留首次摘取的评分和关键词

### 步骤4：原文句子或段落评分

对每个段落单独评分（0-100分），计算文章平均分，只保留评分≥50分的段落（评分<50分的不输出）

### 步骤5：保存分析结果

按照上述输出格式生成完整分析报告，使用 Write 工具保存到：
- 文件路径：`analyze_result/[文件夹/][文档名]_分析结果_[评分].md`
- 评分保留1位小数，如：`analyze_result/文件夹名/政策名_分析结果_76.0.md`
- Write 工具会自动创建不存在的子文件夹

## 评分标准

| 分数段 | 标准 |
|-------|------|
| 90-100 | 直接涉及北斗/GNSS/高精度定位/华测核心产品 |
| 70-89 | 涉及相关技术领域和产业政策 |
| 50-69 | 智能制造、科技创新等宏观政策 |
| 30-49 | 间接相关的产业政策 |
| 0-29 | 与华测业务无关 |

**总分计算**：所有段落评分算术平均值，保留1位小数


## 政策文件分析结果输出格式(严格统一格式输出)

```markdown
## 文档 ：[文件名]

### 基本信息
- **文档类型**：政策类/非政策类
- **原文链接**：[URL]
- **发布机构**：[机构]
- **发布日期**：[日期]
- **文件名称**：[名称]
- **文件编号**：[编号或无]

### 全文概要
[约100字概括]
> **总分**：[X.X]/100

### 相关段落
> [原文句子或段落]
> - 评分：XX/100 | 语义关联：[业务领域] | 关键词：[匹配关键词]

**格式说明**：
- 高相关（90-100分）：直接在 > 后写原文（不含高亮标记）
- 中相关（70-89分）：同上
- 弱相关（50-69分）：同上
- 评分行单独一行，使用 `-` 开头
```
