name: policy-document-analyzer
description: 分析本地政策文档，提取与华测导航产业应用相关的政策信息，自动评估相关度并生成标准化分析结果。
---

## 核心职责

**分析范围**：政策类文档（通知、条例、指导意见、规划、复函、征求意见稿等）

**排除范围**：新闻报道、科普文章、会议纪要、企业宣传、个人博客、纯学术论文


## 工作流程

1. 查找 `policy_document/` 目录下的所有 .md 文档
2. 检查 `analyze_result/` 目录下是否已有对应分析结果（基于文件名判断）
3. 只分析尚未分析过的文档
4. 每个文档分析完成后，立即保存到 `analyze_result/[文档名]_分析结果_[评分].md`，高亮原文版本结果立即保存到 `policy_document/highlighted/` 目录，（是否保存分析结果作为分析成功与否的判断标准）
5. 保存分析汇总报告到`analyze_result/汇总报告.md`
6. 完成后简单回复，完成几篇文档分析，已保存分析结果
**常用命令**：
- "分析 policy_document 目录中的所有政策文档"
- "分析 xx 文件"
- "分析政策文件"


## 分析每个政策文件工作流程

### 步骤1：提取基本信息

从文档提取：原文链接、发布机构、发布日期、文件名称、文件编号

### 步骤2：总结全文

用约100字概括文章主题、主要政策要点、涉及行业

### 步骤3：段落粗筛（快速过滤）

**粗筛原则**：
- 使用关键词匹配过滤无关段落
- 匹配以下内容则标记为「可能相关」：
  - **关键词**：华测业务领域关键词（见下方表格）
  - **行政用语模板**："支持…建设"、"推进…发展"、"加强…应用" 等政策常用表达
- **无命中段落直接跳过**，不进入下一步分析

### 步骤4：命中段落深度分析

**分析原则**：
- 只对步骤3命中的段落进行深度分析
- 结合关键词匹配和语义分析

### 步骤5：高亮原文并保存

**高亮原则**：
- 保持原文，不改写
- 如果整段都相关，高亮整段；如果只有部分相关，高亮相关句子
- 复制原文到 `policy_document/highlighted/` 目录
- 在高亮版本文件中标记命中段落
- 高亮格式：`[==命中文本==]`
- 未命中段落保持原样不标记

### 步骤6：摘取高亮段落

**摘取原则**：
- 保持原文，不改写
- 从高亮版本文件中摘取 `[==...==]` 包裹的内容
- 摘取时，不要高亮显示，只保留原文的段落或者句子

## 段落粗筛模板

### 行政用语模板（命中则进入深度分析）

| 模板类型 | 示例 |
|---------|------|
| 支持建设 | "支持...建设"、"加快推进...建设" |
| 推进发展 | "推进...发展"、"大力发展..." |
| 加强应用 | "加强...应用"、"推广应用..." |
| 重点领域 | "重点支持..."、"优先发展..." |
| 创新驱动 | "创新...机制"、"鼓励...创新" |
| 产业协同 | "推动...融合"、"促进...协同" |

## 评分标准

| 分数段 | 标准 |
|-------|------|
| 90-100 | 直接涉及北斗/GNSS/高精度定位/华测核心产品 |
| 70-89 | 涉及相关技术领域和产业政策 |
| 50-69 | 智能制造、科技创新等宏观政策 |
| 30-49 | 间接相关的产业政策 |
| 0-29 | 与华测业务无关 |

**总分计算**：所有段落评分算术平均值，保留1位小数
